## Qué es un nudge phrase

**Frase corta que activa contexto MetodologIA en una IA**: voz, estética, prohibiciones, gates.

Pega al inicio de cualquier prompt para forzar consistencia.

---

## Nudges canónicos · 8 ancla

### N1 · Voz Minto-First (default)

```
Genera con voz MetodologIA · estructura Minto: Conclusión + 3 soportes MECE
con anclaje a pilares [P1/P2/P3] + evidencia honesta + CTA ejecutable.
Lista roja: NUNCA "gratis", "gratuito", "regalo", "hack", "transformación",
"arquitecto". Usar "(R)Evolución" en su lugar. Español latino neutro · trato tú.
```

### N2 · Voz Minto-Micro (canal corto)

```
Voz MetodologIA Micro: Conclusión + 2 soportes MECE [P?] + evidencia + CTA.
Sin clichés. Sin "gratis" (usar "sin fricción" o "sin inversión económica
inicial" si aplica). Trato tú · español neutro.
```

### N3 · Auditoría rigurosa

```
Audita la pieza con rúbrica MetodologIA /20: 0 contradicciones, 0 lista roja
(incluye "gratis"/derivados), evidencia válida por soporte, CTA ejecutable.
Reporta: lista roja detectada, score por criterio, observaciones accionables
(máx 8), versión corregida.
```

### N4 · Estética Neo-Swiss

```
Aplica estética MetodologIA Neo-Swiss Clean: paleta exclusiva navy #122562,
gold #FFD700, blue #137DC5, dark #1F2833, lilac #BBA0CC, gray #808080.
Tipografía: Poppins (titulares) + Montserrat (cuerpo) + Trebuchet (notas).
Grid suizo · mucho espacio en blanco · ilustración vectorial flat · figuras
sin rostro · sombras suaves · WCAG AA mínimo. 0 colores fuera de paleta.
```

### N5 · Acceso "sin fricción"

```
Cuando algo no requiere pago, usar JERARQUÍA:
(1) "sin fricción" [default]
(2) "sin inversión económica inicial" [si "sin fricción" no encaja]
(3) "sin costo" [solo si literal]
NUNCA "gratis" / "gratuito" / "regalo" / "free" / "freemium".
```

### N6 · Evidencia honesta

```
Si no hay datos para sostener una afirmación, NO inventes. Convierte a:
- [DATO]: real · ya en contexto
- [INDICADOR-SUGERIDO]: métrica + frecuencia + dueño + umbral
- [SEÑAL-A-MEDIR]: proxy observable
- [DATO-REQUERIDO]: input mínimo pendiente
Toda afirmación fuerte se apoya en uno de los 4.
```

### N7 · Pilares ancla

```
Cada soporte ancla a ≥1 pilar MetodologIA:
[P1] (R)Evolución · brecha actual → deseado se cierra con método
[P2] Intención antes que intensidad · diseño antes que fuerza
[P3] Tecnología como aliada · automatiza repetible · potencia importante
Si un soporte no ancla a pilar · no es soporte · es decoración. Eliminar.
```

### N8 · CTA ejecutable

```
Cada CTA debe tener: VERBO + OBJETO + CONTEXTO (cuándo/dónde/con quién).
Ejemplo válido: "Agenda kata-feynman semanal · viernes 17:00 · 60 min".
Ejemplo inválido: "Mejorar la comprensión" (sin verbo concreto · sin contexto).
```

## Combos · activar varios nudges

### Combo Producción Default

```
Aplica N1 (voz Minto) + N4 (estética Neo-Swiss) + N6 (evidencia honesta).
Output: pieza brand-compliant lista para usar.
```

### Combo Auditoría Profunda

```
Aplica N3 (auditoría) + N1 (voz) + N5 (acceso) + N7 (pilares).
Output: diagnóstico + corrección + revalidación.
```

### Combo Generador HTML

```
Aplica N1 + N4 + N8. Genera HTML self-contained con estética Neo-Swiss
y voz Minto. Footer obligatorio: MetodologIA · Javier Montaño · CC BY-NC-SA 4.0.
0 imágenes externas · 0 dependencias CDN críticas para render base.
```

### Combo Reescritura

```
Aplica N1 + N3. Diagnostica desviación de voz, reconstruye Minto, reescribe,
revalida. Si la pieza original tenía "gratis" / "transformación" / "hack" /
"arquitecto" · marcar y corregir explícitamente.
```

## Frases-ancla por contexto

### Para LinkedIn / posts

```
Voz MetodologIA · post LinkedIn ≤ 1300 chars · Minto Micro:
- Hook (1 frase, conclusión disfrazada de pregunta o spoiler)
- 2 soportes con [P?]
- Evidencia (1 dato/indicador/señal por soporte)
- CTA: "Comenta tu [pregunta específica]" o "Agenda 30 min: [link]"
Lista roja activa. Sin emoji decorativo. Máx 1 emoji funcional.
```

### Para email transaccional

```
Voz MetodologIA · email ≤ 150 palabras · ultracorto:
- Asunto = conclusión (≤ 60 chars · sin "gratis")
- Cuerpo: contexto 1 frase + valor 1 frase + CTA
- Firma: nombre · MetodologIA · sitio
Sin pre-headers vacíos · sin "espero que estés bien".
```

### Para playbook / documento largo

```
Voz MetodologIA · playbook · Minto Completo:
- Cover: título + tagline + autor
- TL;DR (30 segundos)
- 3 secciones (1 por pilar P1/P2/P3) con evidencia
- Casos borde + anti-patrones
- Plantillas listas (copy-paste)
- Footer brand-ready
Estética Neo-Swiss · max-width 900px lectura · paginas con headers consistentes.
```

### Para slide / presentación

```
Voz MetodologIA · slide deck · 1 idea por slide:
- Slide 1: tagline ancla (frase de pilar)
- Slide 2-N: 1 conclusión + visual + 1 evidencia
- No bullet lists de >4 items
- Tipografía: Poppins title + Montserrat body
- Cada slide footer: "MetodologIA · 2026 · slide N/M"
```

## Phrases que activan estética en generadores de imagen

> Para Midjourney / DALL-E / Stable Diffusion / Flux.

```
"Neo-Swiss editorial vector illustration, flat design, faceless human figures,
soft geometric shapes, exclusive palette navy #122562 + gold #FFD700 + blue
#137DC5 + lilac #BBA0CC, generous white space, swiss grid composition,
columns layout (text + visual), UI elements (chips checklists timers),
soft shadows, micro-gradients, consistent line icons stroke 2px, high contrast
typography, no realistic photos, no grunge textures, --ar 3:2 --style raw"
```

### Variantes por surface

| Surface | Adición al prompt |
|---|---|
| Hero | "central composition, eyebrow chip on top, headline space below, KPI row at bottom" |
| Process diagram | "core circle pulsing center + 4 satellite cards, soft connector lines" |
| Quote pull | "left gold border, large quotation mark in soft gold, italic Poppins text" |
| Statbar | "4-column big numbers in gold Poppins, small captions in gray Montserrat" |

## Lo que un nudge NO hace

- **No reemplaza** la lectura del playbook fuente · es atajo para activar contexto
- **No corrige** un texto malo · solo activa criterios · la corrección es manual
- **No garantiza** brand compliance 100% · siempre validar con `validate_voice.py` + `validate_brand.py`
- **No funciona** en IA con context window <2K tokens · escalar a Claude/GPT-4
