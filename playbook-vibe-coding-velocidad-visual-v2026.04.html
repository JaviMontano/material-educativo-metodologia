Patrones senior: shipping rápido sin perder rigorSenior patterns: shipping fast without losing rigor

Lo que distingue al developer senior del entusiasta de fin de semana.What distinguishes the senior developer from the weekend enthusiast.

1. AI-Native ADR + Red-Team Review1. AI-Native ADR + Red-Team Review

Antes de generar código, el senior usa la IA para redactar y atacar la decisión arquitectural. Primer pase: la IA propone el ADR (Architecture Decision Record) con constraints duros ("10k req/s, ACID, sin secretos en cliente") y antipatrones explícitos a evitar. Segundo pase: una IA diferente actúa como reviewer escéptico — busca failure scenarios ocultos, asunciones optimistas, dependencias frágiles. DOCBefore generating code, the senior uses AI to draft and attack the architectural decision. First pass: AI proposes the ADR (Architecture Decision Record) with hard constraints ("10k req/s, ACID, no secrets in client") and explicit anti-patterns to avoid. Second pass: a different AI acts as skeptical reviewer — hunts for hidden failure scenarios, optimistic assumptions, fragile dependencies. DOC

2. Hexagonal Architecture (Ports & Adapters)2. Hexagonal Architecture (Ports & Adapters)

La IA escribe mejor cuando el dominio está aislado de la infraestructura. La arquitectura hexagonal permite que el modelo genere entidades de dominio puras sin distraerse con dependencias externas (DB, APIs, queues). Resultado: menos alucinaciones, código más testeable, refactor barato. DOCAI writes better when domain is isolated from infrastructure. Hexagonal architecture allows the model to generate pure domain entities without distraction from external dependencies (DB, APIs, queues). Result: fewer hallucinations, more testable code, cheap refactor. DOC

3. Type-Safe Scaffolding3. Type-Safe Scaffolding

Aprovechar features modernas del lenguaje (TypeScript strict, PHP 8.4 readonly, Rust ownership) como guardrails de generación. Los tipos estrictos previenen errores lógicos durante la generación: la IA no puede escribir código que no compila. DOCLeverage modern language features (TypeScript strict, PHP 8.4 readonly, Rust ownership) as generation guardrails. Strict types prevent logical errors during generation: AI cannot write code that does not compile. DOC

4. CLAUDE.md / .cursorrules — Context persistente4. CLAUDE.md / .cursorrules — Persistent context

Los seniors mantienen un archivo de instrucciones always-on en la raíz del repo. Incluye: stack, conventions, anti-patrones del equipo, comandos de test, deploy, lint. Garantiza que cada generación nueva use el mismo contexto que el build anterior — sin recordatorios manuales. DOCSeniors keep an always-on instruction file at the repo root. Includes: stack, conventions, team anti-patterns, test/deploy/lint commands. Ensures every new generation uses the same context as the previous build — no manual reminders. DOC

5. Tiered prompting (no "one-shot")5. Tiered prompting (no "one-shot")

El senior no pide "construye el sistema entero". Usa una estrategia escalonada: (1) skeleton (modelos + signatures) → (2) guard clauses (validación + errores) → (3) happy path (lógica principal) → (4) tests → (5) static analysis gate. En cada nivel valida antes de avanzar. DOCThe senior does not ask "build the whole system". Uses a tiered strategy: (1) skeleton (models + signatures) → (2) guard clauses (validation + errors) → (3) happy path (main logic) → (4) tests → (5) static analysis gate. Validates at each level before moving on. DOC

6. Progressive rollout obligatorio6. Mandatory progressive rollout

Ningún cambio AI-generado pasa a 100% de tráfico de una. Feature flags + canary (1% → 10% → 50% → 100%) con métricas de rollback automático. La IA acelera desarrollo, no despliegue irreversible. INFERENCIA alineada con la práctica de progressive delivery DOC.No AI-generated change goes to 100% traffic at once. Feature flags + canary (1% → 10% → 50% → 100%) with auto-rollback metrics. AI accelerates development, not irreversible deployment. INFERENCE aligned with progressive delivery practice DOC.
