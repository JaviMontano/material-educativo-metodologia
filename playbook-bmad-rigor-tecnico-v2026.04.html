Por qué BMAD ahoraWhy BMAD now

El vibe-hangover es real, medible y sistémico. La industria se está corrigiendo en 2026.The vibe-hangover is real, measurable, and systemic. Industry is correcting in 2026.

A finales de 2025 la industria reportó el "vibe-hangover": las ganancias iniciales de velocidad fueron compensadas por costos downstream — vulnerabilidades de seguridad (10% del código AI DOC), knowledge ossification (nadie entiende su propio codebase) y hallucination ceiling (el modelo inventa funciones que no existen). Karpathy mismo, autor del término vibe coding, anunció en 2026 la transición a agentic engineering: la IA escribe el 99% del código, pero el 1% humano de orquestación es el factor más crítico para la calidad arquitectural y la mantenibilidad a largo plazo DOC.By late 2025 industry reported the "vibe-hangover": initial speed gains offset by downstream costs — security vulnerabilities (10% of AI code DOC), knowledge ossification (no one understands their own codebase), and hallucination ceiling (model invents non-existent functions). Karpathy himself, author of vibe coding, announced in 2026 the transition to agentic engineering: AI writes 99% of code, but the 1% of human orchestration is the most critical factor for architectural quality and long-term maintainability DOC.

El framework PEV: Plan → Execute → VerifyThe PEV framework: Plan → Execute → Verify

El loop estructurado que reemplaza el prompting informal del vibe coding.The structured loop that replaces vibe coding's informal prompting.
Fase 1Phase 1

Plan

Aquí la expertise de ingeniería es insustituible. Definir objetivo, descomponer en tareas agent-sized, fijar restricciones arquitecturales y especificar quality gates.Here engineering expertise is irreplaceable. Define objective, decompose into agent-sized tasks, set architectural constraints, specify quality gates.

  • Decomposición en tareas atómicasDecomposition into atomic tasks
  • Stack rules + style guidesStack rules + style guides
  • Tests requeridos como gatesRequired tests as gates
  • Boundaries de riesgoRisk boundaries
Fase 2Phase 2

Execute

Agentes especializados trabajan dentro de las restricciones. Implementación, tests, security scan — cada uno por su agente, no uno solo "haciendo todo".Specialized agents work within constraints. Implementation, tests, security scan — each by its agent, not one "doing everything".

  • Implementación → agente Tech LeadImplementation → Tech Lead agent
  • Test suites → agente QATest suites → QA agent
  • Security scan → agente SecOpsSecurity scan → SecOps agent
  • Workflows aisladosIsolated workflows
Fase 3Phase 3

Verify

El humano valida contra el plan. Cambia "escribir código" por "leer y validar código": vulnerabilidades, consistencia arquitectural, cobertura significativa.Human validates against the plan. "Write code" shifts to "read and validate code": vulnerabilities, architectural consistency, meaningful coverage.

  • Diff completo leídoFull diff read
  • Tests significativos verificadosMeaningful tests verified
  • Security review humanoHuman security review
  • Aprobación HITL antes de mergeHITL approval before merge

Model Context Protocol — el "USB-C para IA" que hace posible BMAD a escala enterprise.Model Context Protocol — the "USB-C for AI" that makes BMAD enterprise-scale possible.

Anthropic liberó MCP en noviembre 2024. Para 2026 es el estándar de facto adoptado por OpenAI, Google y Microsoft DOC. Permite conectar agentes a datos (archivos, DBs), tools (APIs) y workflows con una interfaz única estandarizada. Sin MCP, cada agente requiere integración custom para cada fuente. Con MCP, plug-and-play.Anthropic released MCP in November 2024. By 2026 it is the de-facto standard adopted by OpenAI, Google, and Microsoft DOC. Allows connecting agents to data (files, DBs), tools (APIs), and workflows with a single standardized interface. Without MCP, each agent needs custom integration per source. With MCP, plug-and-play.
