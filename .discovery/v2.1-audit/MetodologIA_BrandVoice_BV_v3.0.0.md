## Resumen ejecutivo (decisión en 30 segundos)

**Si quieres voz MetodologIA consistente:** usa **Minto** (Completo o Micro) + **evidencia honesta** + **CTA ejecutable**.

**Regla de oro:** *si una frase no ayuda a decidir, comprender o actuar, sobra.*

| Atajo correcto | Cuándo |
|---|---|
| **Minto Micro** | Chat / poco tiempo |
| **Minto Completo** | Impacto alto · ambigüedad · varias personas |
| **Modo Auditoría** | "Revisión dura" requerida |

---

## Conclusión
MetodologIA diseña contenido de cara a la persona usuaria que **decide**, **opera** y **se mide**, sin promesas infladas: estructura Minto + evidencia verificable (o instrumentable) + CTA de un paso, con lenguaje de diseño y sistemas.

## 3 soportes (MECE)

1. **Decisión primero · [P1]** — una estructura persistente evita contenido "correcto" pero inoperable.
2. **Veracidad sin fricción · [P2]** — lo que no se sabe se declara y se instrumenta; no se rellena.
3. **Escala y consistencia · [P3]** — rúbrica + trazabilidad ligera + SOP permiten repetir calidad y mejorar con tecnología aliada.

## Evidencia
- **S1:** *Criterios de aceptación* + tests MECE.
- **S2:** *Regla de evidencia mínima* + "dato requerido / indicador / señal".
- **S3:** *Rúbrica /20* + métricas de retrabajo y consistencia.

## CTA
Escribe una línea con: **Tarea + Canal + Audiencia + Objetivo + CTA + Modo**. Con eso, este sistema produce.

## 1. Pilares (motor conceptual)

| Pilar | Significado | Soportes típicos |
|---|---|---|
| **P1 · (R)Evolución** | Brecha estado actual → estado deseado se cierra con método. | Diagnóstico claro, diseño de sistema, cadencia, continuidad. |
| **P2 · Intención antes que intensidad** | Diseño antes que fuerza. | Foco, trade-offs explícitos, reducción de fricción, cadencia sostenible. |
| **P3 · Tecnología como aliada** | Automatiza lo repetible; potencia lo importante. | Plantillas, instrumentación, automatización de medición, reducción de retrabajo. |

**Regla:** si un soporte no ancla a un pilar, no es soporte: es decoración.

## 2. Gates no negociables (G1-G6)

- **G1 · Minto persistente** — Conclusión + soportes MECE + evidencia + CTA. Default Completo; Micro si el canal no permite 3 soportes.
- **G2 · Evidencia honesta** — si no hay datos: no inventes. Convierte a *Dato requerido* / *Indicador sugerido* / *Señal a medir*.
- **G3 · Español latino neutro** — sin regionalismos, trato `tú` por defecto.
- **G4 · Lista roja cero tolerancia en piezas finales:** `hack`, `truco`, `secreto`, `transformación`, `arquitecto`, `arquitectura`, `resultados instantáneos`, regionalismos / modismos locales.
- **G5 · Veracidad de fuentes** — no afirmes lectura de fuentes externas si no fueron compartidas.
- **G6 · Anclaje a pilares** — cada soporte ancla explícitamente a al menos 1 pilar (P1-P3).

## 3. Lista verde (preferir)

`método` · `diseñar` · `diseño` · `diseñadores` · `sistemas` · `gobernanza` · `capacidades` · `(R)Evolución` · `Success as a Service` (B2B).

## 4. Definiciones operables (con tests)

### 4.1 Conclusión
**Pasa si:** responde "¿Qué hago ahora?" sin leer el resto.
**Falla si:** describe ("sería bueno…") sin dirección.

### 4.2 Soportes (MECE)
- **Exclusión:** no son reformulaciones.
- **Exhaustividad:** si quitas uno, la conclusión queda incompleta.
- **No fusionables:** si puedes unir dos sin perder significado, fallan.

### 4.3 Evidencia
- **Dato real:** del usuario o contexto compartido.
- **Indicador sugerido:** métrica con definición + frecuencia + dueño + umbral.
- **Señal a medir:** proxy observable.
- **Dato requerido:** input mínimo para afirmar.

### 4.4 CTA
**Pasa si:** verbo + objeto + contexto (cuándo / dónde / con quién, si aplica).
**Falla si:** genérico ("mejorar", "alinear") sin acción concreta.

## 5. Rúbrica /20 (10 criterios × 0/1/2)

1. **Fundamento** — causalidad clara entre conclusión y soportes.
2. **Veracidad** — evidencia válida por soporte; sin afirmaciones huérfanas.
3. **Calidad** — sin contradicciones ni ambigüedades críticas.
4. **Densidad útil** — cada frase aporta decisión / comprensión / acción.
5. **Simplicidad** — mínimo necesario sin perder precisión.
6. **Claridad** — términos definidos cuando importan.
7. **Precisión** — condiciones y límites explícitos.
8. **Profundidad** — causas y trade-offs; no solo recomendaciones.
9. **Coherencia** — estructura y voz consistentes.
10. **Valor** — se puede actuar hoy y medir progreso pronto.

**Aprobación:** 20/20 o corregir y revalidar.

## 6. SOP

### SOP-A · Generar
1. Entradas mínimas (1 línea).
2. Elegir modo (Producción / Auditoría / Chat).
3. Esqueleto Minto (Micro o Completo).
4. Redacción (1-3 patrones máximo).
5. Evidencia por soporte.
6. CTA.
7. Validación (Checklist + lista roja).

### SOP-B · Reescribir
1. Diagnóstico de desviación.
2. Reconstruir Minto.
3. Reescritura.
4. Validación.

### SOP-C · Auditar
1. Contexto.
2. Escaneo lista roja.
3. Checklist + rúbrica /20.
4. Observaciones accionables (máx. 8).
5. Versión corregida.
6. Revalidación.

## 8. Criterios de aceptación

Una pieza queda aprobada si:
- Usa Minto (Micro o Completo) según modo.
- Soportes MECE.
- Evidencia honesta por soporte.
- Lista roja en cero; español neutro tú.
- CTA ejecutable en una acción.
- En Auditoría: rúbrica 20/20.
