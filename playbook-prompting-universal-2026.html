Los 5 pilares del promptingThe 5 pillars of prompting

Si tu prompt cumple estos 5, el 80% del trabajo está hecho. Memorízalos: Claridad · Contexto · Especificidad · Formato · Ejemplos.If your prompt nails these 5, 80% of the work is done. Memorize them: Clarity · Context · Specificity · Format · Examples.

1

ClaridadClarity

Verbo en imperativo. Una instrucción por frase. Sin ambigüedades. Si lo lees y no sabes qué pide, la IA tampoco.Imperative verb. One instruction per sentence. No ambiguities. If you read it and don't know what it asks, neither does the AI.

2

ContextoContext

¿Quién pregunta? ¿Para qué? ¿Qué decisión sigue? El contexto hace que el output sea relevante en vez de genérico.Who's asking? For what? What decision follows? Context makes the output relevant instead of generic.

3

EspecificidadSpecificity

"Resume" es vago. "Resume en 3 viñetas accionables para mi jefe el lunes" es específico. Incluye dimensiones: tamaño, audiencia, deadline."Summarize" is vague. "Summarize in 3 actionable bullets for my boss Monday" is specific. Include dimensions: size, audience, deadline.

4

FormatoFormat

Tabla con columnas X/Y/Z. Markdown. JSON. CSV. Decir el formato evita un round-trip de "ahora ponelo en tabla".Table with columns X/Y/Z. Markdown. JSON. CSV. Stating format avoids a "now put it in a table" round-trip.

5

EjemplosExamples

Few-shot: 2-3 ejemplos del output deseado. Le ahorra a la IA inferir el patrón. Calidad sube ~30%.Few-shot: 2-3 examples of the desired output. Saves the AI from inferring the pattern. Quality up ~30%.

Reduces varianza al máximo si pegas 2-5 ejemplos concretos antes del pedido real. SPEC es el acrónimo del cuerpo de cualquier prompt profesional: Situación · Pedido · Ejecución · Criterios. Antes del SPEC va el encabezado: Nombre, Inputs con notación {[INPUT]}, Resumen. Esa estructura completa es la que sostiene los 2026 prompts de la biblioteca.SPEC is the acronym for the body of any professional prompt: Situation · Petition · Execution · Criteria. Before the SPEC comes the header: Name, Inputs with {[INPUT]} notation, Summary. That complete structure is what sustains the 2026 prompts in the library.

A Encabezado · qué prompt es y qué necesitaHeader · what prompt this is and what it needs

1

Nombre [P3]Name [P3]

Título corto y específico que identifica el prompt. Debe permitir invocarlo después: SWOT v3.4, Plan de comunicación interna, Análisis competitivo Porter. Sin nombre, no se puede catalogar ni reusar.Short specific title that identifies the prompt. Must allow invoking it later: SWOT v3.4, Internal comm plan, Porter competitive analysis. Without a name, it can't be catalogued or reused.

2

Inputs [P2]Inputs [P2]

Variables que el usuario rellena al invocar. Notación canónica: {[NOMBRE_INPUT]} — llave + corchete, mayúsculas, snake_case. Ej: {[ORGANIZACION]}, {[OBJETIVO_NEGOCIO]}, {[HORIZONTE_TIEMPO]}. Cada input declarado debe usarse en EJECUCIÓN o CRITERIOS.Variables the user fills when invoking. Canonical notation: {[INPUT_NAME]} — brace + bracket, uppercase, snake_case. E.g.: {[ORGANIZATION]}, {[BUSINESS_GOAL]}, {[TIME_HORIZON]}. Every declared input must appear in EXECUTION or CRITERIA.

3

Resumen [P1]Summary [P1]

1-2 frases que dicen qué hace el prompt y qué outcome entrega. Útil para que el usuario decida si invocarlo sin leer el SPEC completo. Es el "elevator pitch" del prompt.1-2 sentences saying what the prompt does and what outcome it delivers. Lets the user decide whether to invoke without reading the full SPEC. The prompt's "elevator pitch".

B Cuerpo SPEC · Situación · Pedido · Ejecución · CriteriosSPEC body · Situation · Petition · Execution · Criteria

S

Situación [P2]Situation [P2]

Contexto de uso del prompt. Quién es el usuario, qué problema resuelve, qué decisiones se desprenden del output. Aquí entra también el ROL específico que asume la IA. Sin situación, output genérico.Context where the prompt is used. Who the user is, what problem it solves, what decisions follow from the output. The specific ROLE the AI takes lives here too. Without situation, generic output.

P

Pedido [P2]Petition [P2]

Verbo en imperativo + objeto + propósito + audiencia. "Diseña X para lograr Y, dirigido a Z". Es el corazón del prompt — todo lo demás existe para que el pedido se ejecute con calidad.Imperative verb + object + purpose + audience. "Design X to achieve Y, targeted at Z". The heart of the prompt — everything else exists so the petition executes with quality.

E

Ejecución [P3]Execution [P3]

Pasos numerados que la IA debe seguir para producir el output. Detalla el método, no solo el resultado. Aquí cada {[INPUT]} declarado en el encabezado se usa explícitamente. Sin pasos, la IA improvisa.Numbered steps the AI must follow to produce the output. Describes the method, not just the result. Each {[INPUT]} declared in the header is used explicitly here. Without steps, the AI improvises.

C

Criterios [P1]Criteria [P1]

Cómo se sabe que el output salió bien. Se descompone en 3 sub-bloques: Criterios de Aceptación · Casos Borde · Comprobaciones. La IA usa los Criterios para auto-evaluarse antes de entregar.How you know the output came out right. Decomposes into 3 sub-blocks: Acceptance Criteria · Edge Cases · Checks. The AI uses Criteria to self-evaluate before delivering.

🎯 Sub-descomposición de Criterios [P1]🎯 Criteria sub-decomposition [P1] · Esto es lo que diferencia un prompt amateur de uno profesional: Criterios de Aceptación (lo que debe cumplir el output sí o sí) · Casos Borde (escenarios límite donde el output puede fallar y cómo manejarlos) · Comprobaciones (lista de chequeo que la IA aplica antes de entregar — auto-evaluación). Si los tres sub-bloques están explícitos, la IA no entrega hasta que los pasa todos. Eso es el 100 Check (§ 11) por construcción.This is what separates an amateur prompt from a professional one: Acceptance Criteria (what the output must meet, no exceptions) · Edge Cases (boundary scenarios where output may fail and how to handle them) · Checks (checklist the AI applies before delivering — self-evaluation). If all three sub-blocks are explicit, the AI does not deliver until it passes all of them. That's the 100 Check (§ 11) by construction.

Frente a un problema no-trivial, una IA bien operada no responde de un solo tirón.
